## Pixel Event Mapping

| User Action | Meta Pixel Event | CAPI Event | Optimization Signal |
|-------------|-----------------|------------|---------------------|
| Lands on calculator page | `PageView` (auto) | — | — |
| Starts entering data | `ViewContent` | `ViewContent` | Engaged visitor |
| Gets savings result | `Lead` (custom) | `Lead` | Calculator completed |
| Submits phone number | `Lead` (standard) | `Lead` | Qualified lead |

### Optimization Event Selection
- **Optimize for**: `Lead` event (standard)
- **This fires when**: User submits their phone number after seeing results
- **Why**: Optimizing for the deepest funnel event gives Meta the best signal
- **Learning phase**: Need ~50 Lead events/week for exit (5-7 days at target volume)

## Daily Optimization Routine (15 Minutes)

### Morning Check (10:00 AM)

```
DAILY META ADS ROUTINE
══════════════════════

□ Step 1: Check yesterday's performance (5 min)
  - Open Ads Manager → Campaigns → Last 1 day
  - Note: Spend, Impressions, Clicks, CTR, CPC, Conversions, CPL
  - Compare CPL to target (₹100-₹150)

□ Step 2: Ad Set level check (5 min)
  - Which city has lowest CPL? → Consider scaling
  - Which city has highest CPL? → Consider pausing
  - Any ad set with 0 conversions? → Check if learning limited

□ Step 3: Ad level check (3 min)
  - Which creative angle is winning?
  - Any ad with CTR < 1%? → Consider replacing
  - Any ad with relevance score < 5? → Pause

□ Step 4: One action (2 min)
  - Scale, pause, or test (never all three at once)
```

### Key Metrics to Monitor

| Metric | Good | OK | Bad | Action |
|--------|------|-----|-----|--------|
| CTR (link) | > 2% | 1-2% | < 1% | Bad → New creative |
| CPC | < ₹15 | ₹15-₹30 | > ₹30 | Bad → Adjust audience |
| CPL | < ₹100 | ₹100-₹200 | > ₹200 | Bad → Pause ad set |
| Frequency | < 2 | 2-3 | > 3 | Bad → Expand audience |
| Relevance | 7-10 | 5-6 | 1-4 | Bad → New creative |

## Scaling Rules

### When to Increase Budget

| Condition | Action | Max Increase |
|-----------|--------|--------------|
| CPL < ₹100 for 3 days | Increase ad set budget 20-30% | +30% per change |
| CPL < ₹80 for 5 days | Duplicate ad set to new audience | Double spend |
| Winning creative found | Create variations of winner | 3-5 variations |
| 50+ leads in a week | Launch lookalike audience | ₹200/day |

### Scaling Best Practices
- Never increase budget more than 30% at once (resets learning)
- Wait 3 days after any change before evaluating
- Scale by duplicating winning ad sets, not just increasing budget
- Horizontal scaling (more ad sets) > Vertical scaling (more budget per ad set)

### Lookalike Audiences (After 50+ Leads)

```
Source: Custom Audience from lead list (phone numbers)
Lookalike 1%: Most similar to converters (smallest, highest quality)
Lookalike 2%: Broader (use after 1% saturates)
Lookalike 5%: Broadest (use with strong creative)
```

## Kill Rules (When to Pause)

### Ad Level Kill Rules

| Condition | Wait Period | Action |
|-----------|-------------|--------|
| 0 clicks after ₹500 spent | Immediately | Pause ad |
| CTR < 0.5% after 1,000 impressions | 2 days | Pause ad |
| CPC > ₹50 | 2 days | Pause ad |
| 0 conversions after ₹1,000 spent | 3 days | Pause ad |

### Ad Set Level Kill Rules

| Condition | Wait Period | Action |
|-----------|-------------|--------|
| CPL > ₹300 after 3 days | After learning phase | Pause ad set |
| 0 conversions after ₹2,000 spent | After 5 days | Pause ad set |
| Frequency > 3.5 | Weekly check | Refresh creative or pause |
| Audience exhaustion (reach > 50%) | Weekly check | Expand or new audience |

### Campaign Level Kill Rules

| Condition | Action |
|-----------|--------|
| Overall CPL > ₹250 after 2 weeks | Pause campaign, reassess strategy |
| Lead quality poor (0 connected calls from 20+ leads) | Pause, fix targeting |
| Budget exhausted with < 50% target leads | Reduce budget, optimize first |

## Compliance & Policy Notes

### Meta Ads Policy for Financial Services (India)
- ✅ Allowed: Loan comparison, refinance calculator, financial education
- ✅ Allowed: General savings claims with "up to" or "average" qualifiers
- ❌ Not allowed: Guaranteed approval claims
- ❌ Not allowed: Specific interest rate promises without disclaimers
- ⚠️ Required: "Terms and conditions apply" disclaimer
- ⚠️ Required: Privacy policy link on landing page

### Recommended Disclaimers
- Ad level: "Savings vary based on existing loan terms. Calculator provides estimates only."
- Landing page footer: "MotoFin is a loan comparison platform. Final rates depend on bank approval. T&C apply."
